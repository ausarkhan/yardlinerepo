YardLine Playwright Commerce QA Report
======================================

Run date: 2026-06-30
Branch: playwright-commerce-qa

Scope
-----

This sprint added a Playwright-powered live commerce QA suite for the deployed YardLine frontend and backend:

- Frontend: https://yardlinerepo.vercel.app
- Backend: https://yardlinerepo-production.up.railway.app
- Health: https://yardlinerepo-production.up.railway.app/health

The suite intentionally uses Stripe test card data only. QA-created records are prefixed with `PLAYWRIGHT COMMERCE QA [timestamp]`.

How To Run
----------

From `webapp/`:

```bash
npm run test:commerce:auth
npm run test:commerce:paid-ticket
npm run test:commerce:booking
npm run test:commerce:live
STRIPE_SECRET_KEY=sk_test_... npm run test:commerce:stripe-session -- cs_test_...
```

Headed mode is opt-in:

```bash
PLAYWRIGHT_HEADED=1 npm run test:commerce:live
```

OTP handling:

- Auth setup is separated from commerce validation in `webapp/e2e/live-commerce.auth.setup.ts`.
- Creator auth is saved to `webapp/playwright/.auth/creator.json`.
- Buyer/admin auth is saved to `webapp/playwright/.auth/buyer.json`.
- The auth setup validates saved state before prompting. OTP is requested only when the state file is missing, expired, or redirects back to login/verify.
- The test prints `Please provide the OTP for [email].` only during auth refresh.
- It accepts stdin when available.
- It also watches `/tmp/yardline-otp-[email].txt`; write the 6-digit code there if stdin is not consumed by the Playwright worker.
- `CREATOR_OTP` and `BUYER_OTP` environment variables are also supported for one-off local runs.

Bugs Fixed In This Branch
-------------------------

- Commerce auth is no longer embedded as the first serial commerce test.
- Added separate npm scripts:
  - `npm run test:commerce:auth`
  - `npm run test:commerce:paid-ticket`
  - `npm run test:commerce:booking`
  - `npm run test:commerce:live`
- Paid ticket quantity selection now scopes to the ticket tier card instead of accidentally selecting the disabled Buy Tickets button.
- Paid ticket checkout waits across the legal gate instead of asserting on a removed/hidden Buy Tickets button.
- Stripe Checkout submit now avoids hidden accordion buttons and targets the visible exact `Pay` button.
- Stripe Checkout submit now opts out of Link saved-info so Stripe does not require a phone number during test card payment.
- Stripe AI-agent disclosure handling is applied before hosted Checkout submit and embedded booking authorization.
- Embedded Stripe billing name/ZIP fields are optional so booking works when Stripe Elements omits them.
- Booking status assertions are scoped to the new booking/confirmation instead of matching old page history.
- Added `webapp/scripts/validate-stripe-checkout.mjs` and `npm run test:commerce:stripe-session` to retrieve Checkout Session, expanded PaymentIntent, related Stripe test events, and a state classification when `STRIPE_SECRET_KEY` is available.
- Auth setup now re-saves storage state after successful validation so refreshed Supabase tokens are persisted for later API checks.
- Playwright SPA navigation helpers now route through the mounted Vercel app so direct client paths do not depend on platform rewrites during tests.
- OTP entry now supports the deployed auto-submit OTP component.
- OTP prompts now support stdin, environment variables, and `/tmp` files for reliable interactive Playwright runs.
- Auth assertions were tightened to avoid strict locator collisions.
- Live Playwright config now defaults to headless mode with `PLAYWRIGHT_HEADED=1` opt-in.
- Test diagnostics now capture browser console, network, HTTP status, screenshots, traces, and videos.
