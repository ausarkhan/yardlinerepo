# YardLine — Phase 5 Summary: Trust, Safety, Messaging, Reviews, Admin & Compliance

Phase 5 turns the marketplace into a trusted campus platform. Everything runs through the
webapp talking directly to Supabase under RLS (the backend is not the data path here).

> **Database apply step:** Phase 5 ships one additive, non-breaking migration. Apply it once
> in the Supabase SQL editor (or `supabase db push`):
> `supabase/migrations/0004_phase5_trust_safety.sql`.
> Reviews work against the existing `reviews` table immediately; messaging, reports,
> notifications, waivers, and admin moderation activate the moment the migration is applied.
> No existing tables/columns were renamed or dropped. No mock data.

## New routes
| Route | Page | Notes |
|-------|------|-------|
| `/messages` | Messages | 1:1 conversation list, unread badges |
| `/chat/:id` | Chat | Realtime thread, read receipts, report user |
| `/report` | Report | Submit/report history (`?type=&id=` prefill) |

| `/terms`, `/privacy`, `/waiver` | Legal | Public legal documents + versions |

## New tables (migration 0004)
- **conversations** — 1:1 ordered participant pair + context (provider/booking/event), last-message preview.
- **messages** — thread messages with `read_at`; realtime-published.
- **waiver_acceptances** — `document` (terms/privacy/waiver) + `waiver_version` + `accepted_at`; supports re-acceptance.
- **reports** — target_type/target_id/category/status; reporter + admin RLS.
- **moderation_actions** — immutable audit log (admin_id, action, target, reason, metadata).
- **notifications** — per-user, realtime; generated by SECURITY DEFINER triggers.
- **profiles** (extended) — `role` (user/admin), `status` (active/suspended), `suspended_at`, `suspension_reason`.
- **`is_admin()`** SECURITY DEFINER helper gating all admin RLS policies.
- **Triggers** generate notifications on new message, booking status change, new review, report resolution.

## Subsystems delivered
- **Messaging** — start from provider/event/booking/chat; Supabase realtime; per-conversation + global unread; mark-read.
- **Reviews (1–10)** — eligibility-gated (completed booking / used ticket); averages + recent reviews on provider & event pages.
- **Reputation** — trust indicators (avg rating, completed bookings, events hosted, attendees served) on provider/event pages.
- **Waiver/legal gate** — `useWaiverGate` blocks becoming a provider, creating a paid event, booking, and buying a ticket until Terms+Privacy+Waiver are accepted.
- **Reporting & moderation** — report any event/provider/user/message; admin queue with dismiss/action + required reason.
- **Admin dashboard** — platform metrics; user suspend/reinstate; event hide/restore; provider disable/restore (soft deletes via existing `is_hidden`); moderation history.
- **Notifications** — nav bell with realtime unread; message/booking/review/report/admin types.

## Permission model
- **Admin** (`profiles.role='admin'`) — RLS-gated moderation across users/events/providers/reports; first admin seeded by email.
- **Reporter** — insert + read own reports only.
- **Conversation participants** — read/write only their own threads.
- **All moderation writes** require a reason and are logged to `moderation_actions`; nothing is hard-deleted.
